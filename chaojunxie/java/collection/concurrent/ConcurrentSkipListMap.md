## ConcurrentSkipListMap(对应TreeMap)

+ 跳表

  跳表是通过空间换时间的概念，来实现快速的查找，它的查找效率和红黑树接近

  + 结构
    1. 跳表的最下层都是一个有序的单向链表
    2. 在这个有序链表的基础上，跳表使用了多层索引结构，每一层的元素都来源于上一层，但是会比上一层的元素要少
    3. 每个结点有两个指针，一个指向同一层的下一个结点，另外一个指向下一层的结点

  + 查找

    查找思路整体流程和二分法类似，从最高层索引开始，和下一个结点对比，比下个结点大就右移动，反之就下移动

  + 插入

    跳表的插入，必定会在最底层的单向链表插入记录，但是别的层次不保证。会插入到多少层，是通过概率计算出一个值来表明会插入到多少层索引当中

  + 删除

    找到该节点以后，在各层索引中删除

+ 实现原理

  操作整体都是通过循环+CAS的方式来实现跳表的算法

+ 特点

  1. 没有使用锁，所有操作都是无阻塞的，所以操作都可以并行
  2. 实现了ConcurrentMap接口，支持了一些原子性的操作
  3. 和TreeMap一样，是可排序的
